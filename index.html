🎯 El flujo · cómo se entrena un escenario completo

🔴
PASO 1 · RED TEAM

Lanza un ataque desde la Console tab. Elige un playbook (ej. apt29-cozy-bear) y ejecútalo contra un target del lab. Los pasos se ejecutan en tiempo real con streaming WebSocket.

🔵
PASO 2 · BLUE TEAM

Verifica en Wazuh dashboard si la regla de detección disparó. Cada paso del playbook documenta el defense_signal esperado: regla Sigma, sysmon event ID, suricata SID. Si no apareció = gap de detección.

💜
PASO 3 · PURPLE TEAM

En Purple Team tab, marca el run como ✓ detected o ✗ missed. La matriz MITRE se actualiza · ves qué técnicas tienes cubiertas vs gaps. Itera: ajusta reglas → re-ejecuta → mide mejora.

📄
PASO 4 · REPORT

Genera reporte ethical hacker desde Console → History → 📄 Report. HTML profesional con executive summary, MITRE coverage, recommendations + Atomic Red Team refs. Exportable.

🧠 Jarvis · Panel operativo completo

¿Qué es Jarvis?

Jarvis es un orquestador de IA local-first diseñado para operar el cyber range, automatizar tareas de análisis, defensa y entrenamiento sin gastar tokens cloud. Es el cerebro operativo de toda la plataforma: cada solicitud se enruta primero a modelos locales Ollama; sólo si el modelo local no puede resolverla escala a Claude/Gemini como fallback.

Arquitectura zero-token: 13 modelos Ollama desplegados (qwen3:14b, qwen2.5-coder:7b, qwen2.5:7b, gemma-2-fast, llama3.1, nomic-embed-text, etc.) ejecutados en GPU 8GB. Circuit breaker en el proxy Ollama (3 fallos/30s → open 120s) más routing por carga garantiza disponibilidad bajo presión.

Capacidades operativas:

  • Generación de código · TypeScript/React/Python/Bash/PowerShell con qwen2.5-coder:7b
  • Análisis de logs y forense · ingesta Wazuh/Suricata/Velociraptor + razonamiento con qwen3:14b
  • RAG semántico · 3 stores (PostgreSQL+pgvector, Qdrant, SQLite) sobre 2029+ chunks · embeddings nomic-embed-text 768-dim
  • Traducción multilingüe · ES/EN/FR/PT sin coste de API · usado por el dashboard para i18n on-the-fly
  • Orquestación multi-agente · agentes especializados (Recon, Exploit, PostExploit, Defense, Reporter) coordinados vía MCP sobre OpenClaw
  • Ejecución de skills · 70+ skills cargadas (cybersec, dev, scraping, deploy) con dispatcher LLM
  • Generación de reportes ethical hacker · executive summary + findings + remediation con plantillas tipo CEH

Stack: Node HTTP server :7777 · Chrome --app · Ollama proxy con OTEL :11435 · sesiones persistentes en %LOCALAPPDATA%/jarvis-desktop/sessions/ · atajo VSCode Ctrl+Alt+J · disponibilidad 24/7 con health monitor + auto-remediation.

  • defense signal — qué debería detectar el blue team
